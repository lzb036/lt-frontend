# 商品采集系统使用手册

> 适用对象：商品操作员、店铺操作员和系统管理员
>
> 适用范围：商品采集、审核、上架、店铺商品、订单同步、销量筛选和任务处理
>
> 不包含：用户管理、敏感词管理、代理配置等独立管理员功能

## 1. 操作前须知

系统中的操作分为三类：

| 类型 | 说明 | 典型操作 |
| --- | --- | --- |
| 仅修改系统数据 | 不会立即修改乐天店铺 | 采集、编辑待审核商品、审核 |
| 会修改乐天店铺 | 提交后会写入真实店铺 | 上架、同步修改、上下架、替换、删除 |
| 混合操作 | 页面本身主要查看数据，部分按钮会写入乐天 | 已审核商品、同步任务 |

执行会修改乐天店铺的操作前，必须确认：

1. 当前选择的店铺正确。
2. 商品管理编号正确。
3. 商品标题、价格、SKU、品类和图片已经检查。
4. 没有相同任务正在等待或运行。

## 2. 标准工作流程

普通商品操作员的标准流程如下：

1. 使用“手动采集”，或等待采集店铺的定时任务完成。
2. 在“待审核商品”中检查并编辑商品。
3. 点击“保存”后执行“审核通过”。
4. 在“已审核商品”中选择目标店铺创建上架任务。
5. 在“上架任务”中确认执行结果。
6. 在“已上架商品”中检查店铺标签。
7. 在“店铺商品”中检查乐天店铺中的实际商品。

## 3. 手动采集

进入路径：`任务管理 > 手动采集`

1. 选择采集方式。
2. 填写有效的乐天商品链接或目标来源。
3. 提交采集。
4. 等待任务完成并查看成功数、失败数和失败原因。
5. 采集成功的商品会进入“待审核商品”。

注意事项：

- 不要重复提交同一个商品链接。
- “商品未保存”通常表示商品缺少必要字段、已经存在或不符合保存条件。
- 部分成功时，应展开失败明细逐条处理。

## 4. 定时采集与采集店铺

进入路径：

- `店铺管理 > 采集店铺`
- `任务管理 > 定时采集`

操作步骤：

1. 在“采集店铺”中添加来源店铺并设置采集范围。
2. 启用后，系统会按计划创建采集任务。
3. 临时需要采集时可以使用“立即执行”。
4. 在“定时采集”中查看任务状态和结果。
5. 不再需要自动采集时停用来源店铺。

停用来源店铺只影响后续自动调度，不影响已有任务记录，也不限制临时的手动立即执行。

## 5. 待审核商品

进入路径：`商品管理 > 待审核商品`

### 5.1 查看和编辑

1. 使用关键词、采集时间等条件找到商品。
2. 点击“查看详情”。
3. 检查标题、副标题、品类、价格、SKU 和图片。
4. 完成修改后点击“保存”。

只有点击“保存”按钮才会保存修改。关闭详情弹窗不会自动保存。

### 5.2 审核前检查

- 标题不能为空。
- 价格必须是大于零的日元整数。
- 必须存在有效的 SKU。
- 必须至少存在一张有效商品图片。
- 品类必须通过系统品类选择器选择。
- 品类编号必须是有效的六位编号。

审核通过后，商品进入“已审核商品”，但不会自动上架。

## 6. 已审核商品

进入路径：`商品管理 > 已审核商品`

已审核商品已经完成内容审核，等待选择店铺上架。

1. 打开详情复核标题、副标题、价格、SKU 和品类。
2. 品类在已审核详情中只读展示，不能修改。
3. 选择一个或多个商品。
4. 点击上架并选择目标店铺。
5. 确认商品数量和店铺数量后创建任务。
6. 前往“上架任务”查看结果。

批量上架前必须再次确认目标店铺，避免将商品上传到错误店铺。

## 7. 上架任务

进入路径：`任务管理 > 上架任务`

任务可能处于以下状态：

- 等待中：已经进入队列，尚未开始执行。
- 运行中：正在上传图片或写入乐天。
- 成功：商品已完成上架。
- 部分成功：批量任务中只有部分商品或店铺成功。
- 失败：任务未完成，需要查看错误详情。
- 已取消：任务在允许取消的阶段被终止。

失败时的处理顺序：

1. 查看具体商品、店铺和错误字段。
2. 修正商品资料或店铺配置。
3. 确认旧任务已经结束。
4. 再创建任务或使用页面提供的重试功能。

## 8. 已上架商品

进入路径：`商品管理 > 已上架商品`

已上架商品从商品维度展示商品已经上架到哪些店铺。

- 店铺标签表示商品已上架到对应店铺。
- 点击店铺标签可以复制该店铺的商品管理编号。
- 一个商品可以关联多个店铺。
- 每个店铺可以拥有不同的商品管理编号。
- “打开源店铺商品页”和“打开店铺商品页”指向不同页面。

需要将商品继续上架到其他店铺时，可以从此页面创建重新上架任务。

## 9. 店铺商品

进入路径：`店铺管理 > 店铺商品`

店铺商品代表乐天店铺中的实际商品。操作前先确认当前店铺。

可以进行的常见操作：

- 查看店铺商品详情。
- 同步修改允许编辑的商品字段。
- 修改上下架状态。
- 执行商品替换。
- 按系统权限和确认流程删除店铺商品。

“同步修改”、上下架、替换和删除都会影响真实乐天店铺。

### 9.1 多周期销量

店铺商品支持按以下周期查看有效销量：

- 近一年。
- 近半年。
- 近3个月。
- 近1个月。
- 近1周。

有效销量口径：

```text
有效销量 = 下单数量 - 取消数量 - 已确认退款数量 - 已确认退货数量
```

销量排序和销量范围都按照当前选择的周期计算。可以选择从高到低、从低到高，也可以填写最小销量和最大销量。系统会先对全部符合条件的商品筛选和排序，再执行分页。

已完成订单首次同步但周期内无销量时显示 `0`；尚未完成首次同步时显示 `-`。

## 10. 商品替换

商品替换用于保留原商品管理编号，同时将该编号下的商品内容替换为新商品。

操作步骤：

1. 在“店铺商品”中找到目标商品。
2. 点击“替换商品”。
3. 输入用于替换的乐天商品链接。
4. 等待采集完成。
5. 检查替换后的标题、副标题、品类、价格、SKU 和图片。
6. 保存替换草稿。
7. 输入目标商品管理编号进行确认。
8. 在“同步任务”中查看执行结果。

替换成功后：

- 乐天店铺继续使用原商品管理编号。
- 原已上架商品的内容不会被覆盖。
- 原已上架商品会移除当前店铺标签。
- 系统会生成新的已上架商品。
- 新已上架商品会关联当前店铺。
- 店铺商品会关联到新的已上架商品。

商品替换属于高风险操作。店铺、替换内容和商品管理编号都确认无误后再提交。

## 11. 同步任务

进入路径：`任务管理 > 同步任务`

同步任务用于跟踪：

- 商品替换。
- 店铺商品同步修改。
- 上下架状态变更。
- 店铺商品删除。
- 其他需要写入乐天的同步操作。

失败时先确认任务是否已经开始写入乐天。网络中断可能发生在乐天已经写入、本地尚未完成记录的阶段，不要直接重复执行。

## 12. 店铺信息与订单数据

进入路径：`店铺管理 > 店铺信息`

- “近一年订单数”表示当前本地保留的滚动365天订单总量。
- 尚未完成首次同步时显示 `-`。
- 已完成同步但没有订单时显示 `0`。
- 新店铺首次订单同步默认获取最近365天。
- 系统会自动清理超过365天的订单，因此总数可能随时间减少。

近一年订单数不是自然年度累计，也不是某一次订单同步处理的订单数量。

## 13. 订单获取记录

进入路径：`任务管理 > 订单获取记录`

所需权限：`超级管理员`。普通用户不会看到此菜单，也不能直接访问该页面。

- 可以按店铺、触发方式、状态和创建时间筛选。
- 店铺列显示店铺别称。
- “订单总数”显示店铺当前近365天订单总量。
- “新增”显示该次同步新增的订单数量。
- 删除订单获取记录不会删除订单、销量数据或同步状态。
- 失败记录可以在确认原因后重试。

自动增量同步没有新订单时，“新增”为 `0` 属于正常情况。

## 14. 订单同步设置

进入路径：`系统设置 > 订单同步设置`

所需权限：`超级管理员`。

管理员可以配置：

- 是否启用自动同步。
- 自动同步间隔，范围为5到1440分钟。
- 成功记录保留天数，范围为1到365天。

关闭自动同步后，手动立即更新仍可使用。成功记录保留只影响任务记录，不会删除订单和销量数据。

该设置是全局统一配置，所有用户和所有店铺共用，不会按用户分别保存。

只有点击“保存”按钮才会应用修改。

## 15. 异常商品与常见问题

进入路径：

- `商品管理 > 异常商品`
- 各任务的错误详情

### 商品缺少有效品类

回到允许编辑商品的阶段，通过品类选择器选择有效品类。不能只填写品类文字。

### 商品标题为空

检查采集结果和敏感内容处理结果，补充有效标题后保存。

### 商品未保存

常见原因：

- 缺少标题或商品链接。
- 商品已经存在。
- 商品已经存在于店铺商品中。
- 采集结果缺少系统要求的必要字段。

### 上架失败

检查：

- 店铺是否启用。
- 店铺凭证是否有效。
- 标题、价格、SKU、品类和图片是否符合要求。
- 乐天错误详情是否指出具体字段。

### 任务长时间等待

不要立即重复创建任务。先确认：

- 是否已经存在相同商品和店铺的任务。
- 任务是否仍处于等待或运行状态。
- 是否存在前序任务尚未完成。

### 无法判断乐天是否已经写入

记录以下信息后联系管理员：

- 任务编号。
- 操作时间。
- 店铺名称。
- 商品管理编号。
- 页面显示的完整错误信息。

## 16. 安全操作原则

1. 修改详情后只有点击保存才会生效。
2. 批量操作前确认商品数量和店铺数量。
3. 涉及乐天的操作前确认当前店铺。
4. 商品替换和删除必须确认商品管理编号。
5. 任务运行期间不要重复提交相同操作。
6. 错误原因不明确时停止重复操作并联系管理员。

## 17. 手册维护

系统内使用手册的结构化内容位于：

```text
src/utils/operatorManual.ts
```

本 Markdown 文件用于培训、审阅和离线查阅。以下变化发生时应同步更新两处内容：

- 页面入口或菜单名称变化。
- 按钮名称或操作步骤变化。
- 权限限制变化。
- 操作结果变化。
- 是否影响乐天店铺发生变化。
- 错误提示或异常处理方式变化。
